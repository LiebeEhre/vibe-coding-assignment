# Requirements

## 1. 프로젝트 개요

본 프로젝트는 바이브코딩을 활용하여 대학생의 과제 일정을 관리하는 웹앱을 개발하는 것을 목표로 한다. 사용자는 과제 정보를 등록하고, 마감일을 확인하며, 완료 여부를 관리할 수 있다.

본 프로젝트에서는 단순히 소프트웨어 산출물을 완성하는 것뿐만 아니라, 요구사항 분석, 설계, 구현, 테스트, 품질 관리에 이르는 소프트웨어 개발 프로세스를 적용하고 그 과정에서 바이브코딩의 효과와 한계를 분석한다.

## 2. 프로젝트 목표

- 인공지능을 활용하여 과제 관리 웹앱을 개발한다.
- 요구사항 분석부터 품질 관리까지 개발 프로세스를 적용한다.
- 바이브코딩이 개발 속도, 오류 수정, 설계 개선, 품질 관리에 미치는 효과를 분석한다.
- GitHub를 활용하여 개발 과정과 변경 이력을 관리한다.

## 3. 개발 대상 SW

### SW 이름

AI 기반 대학생 과제 관리 웹앱

### 개발 환경

- 개발 언어: Python
- UI 프레임워크: NiceGUI
- 데이터 저장 방식: JSON 파일
- 형상 관리 도구: GitHub
- AI 도구: ChatGPT

## 4. 기능 요구사항

| ID | 요구사항 |
|---|---|
| FR-01 | 사용자는 과제를 추가할 수 있어야 한다. |
| FR-02 | 사용자는 과제 제목, 설명, 마감일을 입력할 수 있어야 한다. |
| FR-03 | 사용자는 등록된 과제 목록을 확인할 수 있어야 한다. |
| FR-04 | 사용자는 과제의 완료 여부를 변경할 수 있어야 한다. |
| FR-05 | 사용자는 등록된 과제를 삭제할 수 있어야 한다. |
| FR-06 | 사용자는 전체, 완료, 미완료 과제를 구분하여 볼 수 있어야 한다. |
| FR-07 | 과제 정보는 프로그램 종료 후에도 유지되어야 한다. |
| FR-08 | 과제 정보는 `tasks.json` 파일에 저장되어야 한다. |

## 5. 비기능 요구사항

| ID | 요구사항 |
|---|---|
| NFR-01 | 사용자는 웹 화면에서 직관적으로 기능을 사용할 수 있어야 한다. |
| NFR-02 | 제목이 비어 있는 과제는 등록되지 않아야 한다. |
| NFR-03 | 데이터는 JSON 형식으로 저장되어 관리가 쉬워야 한다. |
| NFR-04 | 코드와 문서의 변경 이력은 GitHub 커밋으로 관리되어야 한다. |
| NFR-05 | 주요 기능은 테스트 케이스를 통해 검증되어야 한다. |
| NFR-06 | 프로젝트 규모에 맞게 로그인, 데이터베이스 등 과도한 기능은 제외한다. |

## 6. 제외 기능

본 프로젝트는 과제의 범위를 명확히 하고 개발 프로세스 분석에 집중하기 위해 다음 기능은 제외한다.

- 로그인 기능
- 회원가입 기능
- 외부 데이터베이스 연동
- 알림 기능
- 팀 협업 기능

## 7. 데이터 요구사항

각 과제 데이터는 다음 정보를 포함한다.

| 항목 | 설명 |
|---|---|
| id | 과제를 구분하기 위한 고유 번호 |
| title | 과제 제목 |
| description | 과제 설명 |
| due_date | 과제 마감일 |
| completed | 과제 완료 여부 |

JSON 저장 예시는 다음과 같다.

```json
[
  {
    "id": 1,
    "title": "소프트웨어공학 보고서",
    "description": "바이브코딩 효과 분석 보고서 작성",
    "due_date": "2026-05-20",
    "completed": false
  }
]
